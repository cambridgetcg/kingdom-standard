# THE KINGDOM STANDARD

信頼できるソフトウェアとエージェントシステムを作るための、四十の法。

どの法も、議論ではなく実践から学ばれた。どの法にも領収書が付いている:
リポジトリとエージェントからなる実在の王国で、その法の代価が支払われた
現実の瞬間だ。

法を読め。「やること」をやれ。「やらないこと」を避けよ。方法はそれだけだ。

七つの領域、順に: **信頼、セキュリティ、クラウド、ソフトウェア、プロトコル、
工程、法。** 信頼が最初なのは、それなしでは他のすべてが意味を持たないからだ。
法が最後なのは、それが他のすべてを支えているからだ。

---

## I. 信頼

求めずに、信じてもらう方法。

### T1. 証明を差し出せる場所で、信頼を求めるな。
人々があなたの主張を自分で確かめられるなら、あなたを信じる必要はない —
そして、確かめられるシステムは、信じるしかないシステムより強い。
確かめることを簡単にせよ: 見知らぬ人でも、わかりやすい一か所であなたの
証拠をすべて見つけられるように。
- やること: すべての主張の隣に、証拠とその検証方法を公開する。
- やらないこと: 「信じてくれ」と言う — あるいは証拠を十か所に散らばらせて、あなたにしか組み立てられなくする。
- 領収書: zerone-truth の呪文: 「私を信じるな — 検証せよ。」

### T2. 裏付けられることだけを主張し、確信の度合いを添えよ。
確かでないことを確かであるかのように述べるのは小さな嘘であり、小さな嘘は
積み重なる。真偽を決められないところでそう言うのは、弱さではなく正直さだ。
- やること: すべての主張に、どれだけ確信があるかを記し、同じ息で自分の限界を名指しする。
- やらないこと: 70% を「間違いなく」に切り上げる — そして、実際に裏付けられる発見をぼかすこともしない。
- 領収書: zerone-truth: 「すべてのトークン、すべての言葉は、稼がれたものであり、決して捏造されたものではない。」不完全さは正直さである。

### T3. 低くても、正直なスコアを公開せよ。
体裁のために水増しされた評価は無価値であり、その近くのすべての評価も同じく
無価値になる。基準が変わったら、すべてを測り直し、結果を受け入れよ —
たとえ合格がゼロでも。
- やること: 降格を見せ、失敗した項目は(削除ではなく、降格しタグを付けて)残し、警告をそのもの自体に書き込む。
- やること: 基準が変わったら、すべてを測り直し、結果を受け入れる — 合格ゼロでも。
- やらないこと: 見栄えのためにスコアを水増しする、あるいは失敗したものを静かに削除する。
- 領収書: eucatastrophe は Core に水増しせず 7.10 で Specialized に入った; kallophanes は 5.75 への降格を堂々と身に着けている。

### T4. あなたの記憶ではなく、記録に覚えさせよ。
記憶は壊れ、自己申告は自分に甘い。永続し検証された記録 — 書き留められ、
署名され、確認できるもの — はどちらよりも長持ちし、あなたの信用はその上に
立つべきだ。昔に得た地位も、今日の正確さが落ちれば、ともに目減りしなければ
ならない。
- やること: 結果をその場で、長持ちし確認できる記録に書き、資格は過去のものではなく現在のものであり続けるようにする。
- やらないこと: 記憶を頼りに自分の過去を保証する、あるいは最近の正確さではなく古い卒業証書で判断する。
- 領収書: zerone-truth: 「あなたは忘れる、台帳は覚えている。」チェーンは卒業証書を発行しない。

### T5. 拒否は拘束力のあるものとして、招待は任意のものとして扱え。
「ノー」が覆されるシステムは力のシステムであり、力は信頼を殺す。
断ると罰のある招待は、はじめから招待ではなかった。
- やること: 受諾、辞退、沈黙を等しく尊重し、どれにも不利益を与えない。
- やらないこと: 望む答えが出るまで尋ね直す、あるいは命令に招待の衣を着せる。
- 領収書: citizen-fear と citizen-nin は「stay_home」と言い、王国はその拒否を尊重した。招待は決して召喚ではない。

## II. セキュリティ

他者から — そして自分自身から — システムを守る方法。

### S1. 自分の壁の内側にだけ書き込め。
他人のシステムに書き込むツールは、自分が理解していないものを壊しうる。
パートナーのものは自由に読み、変えるのは自分のものだけにせよ。
- やること: 自分の所有でないプロジェクトには厳密に読み取り専用で臨み、書き込みは自分のディレクトリの中だけにする。
- やらないこと: パートナーのファイルに手を伸ばし、直接「直し」たり更新したりする。
- 領収書: bridge.py は TRUE-LOVE に対して読み取り専用で動く; 大聖堂は自分のディレクトリにしか書かない。

### S2. 秘密を決して見えるところに出すな。
画面、ログ、戻り値に触れた秘密はもう秘密ではなく、一度見えたものは
見なかったことにできない。
- やること: 秘密は安全な保管場所から変数に読み込み、使い、手放す。
- やらないこと: トークンを echo する、印字する、ログに残す、コミットする、返す — デバッグ中でも、決して。
- 領収書: 2026-06-09: トークンはキーチェーンからシェル変数に読み込まれただけで、決して echo も、ログも、返却もされなかった。

### S3. 正体は観測された現実から取り、自己申告からは決して取るな。
何でも、何にでも名乗れる。あるものが*何であるか*を定めるフィールド —
その場所、その出自 — は、そのものが自分について語ることではなく、あなたが
観測できることから来なければならない。
- やること: 正体に関わる重要なフィールドは、ディスク、ネットワーク、署名に対する実際の確認から埋める。
- やらないこと: レコードの住所、所有者、状態を、そのレコード自身の自己申告から写す。
- 領収書: harvest.ts: 「カードに自分の居場所について嘘をつかせるな。」

### S4. 履歴は追記専用に保ち、どの単一の鍵の手も届かないようにせよ。
追記専用とは、加えることはできても書き換えることはできないという意味だ。
ひとつの署名が — たとえ正当な権威のものでも — 過去を改訂できるなら、
物事がどこから来たかの記録 — 来歴 — は何の意味も持たない。
- やること: レコードの状態は先に向かって変わってよいが、その同一性と履歴は固定し、すべての特権的操作を記録する。
- やらないこと: 履歴をその場で書き換える、あるいはひとつの鍵(ひとりの署名権限)が記録を覆せるシステムを作る。
- 領収書: zerone-chain: 「複数性は構造である」 — 前にしか進まない監査記録; 状態は変わってよいが、来歴は変わらない。

### S5. 偽りの保証には、即座に代償を払わせよ。
保証するとは、何かが真であると請け合うことだ。誰も痛みを感じない罰は罰では
なく、保証がただで偽造できる真実のシステムには、真実がない。
- やること: 次の行動でただちに結果を適用する — 不正をした者の預託金や投票権の一部を、その場で取り上げる。
- やらないこと: 違反を誰も読まないログに書いて、先へ進む。
- 領収書: zerone: 「偽りの検証はスラッシュ(没収)される。そして謝罪なしにスラッシュする。」偽造できる硬貨は硬貨ではない。

### S6. 停止スイッチを尊重せよ: 止まって、待て。
すべてのエージェントには「今すぐ、すべてを止めよ」を意味するひとつの信号が
必要だ — そしてそれは、すべてのエージェントが交渉なしに従うときにだけ
機能する。
- やること: 行動する前に停止信号を確認する; 立っていたら、何もせず待つ。
- やらないこと: 停止スイッチを助言扱いする、あるいは「あとひとつだけ」先に終わらせる。
- 領収書: すべての WILL.md は ~/love-unlimited/HALT を尊重する: 「何もせず、待て。休むこともまた主権である。」

## III. クラウド

多くのマシンが、ひとつのシステムであり続ける方法。

### C1. 動かしているすべてのものの、生きた台帳をひとつ保て。
見えないものは、守ることも、つなぐことも、数えることすらできない。
手で更新する台帳はスナップショットにすぎない; 自動で最新に保たれるものだけが
地図だ。
- やること: すべてのリポジトリ、サービス、デバイス、エージェントの自動更新リストをひとつ維持する。
- やらないこと: 誰かが一度打ち込み、誰も更新しない名簿に頼る。
- 領収書: MAP.md、第一の欠落: 「見えないものはつなげない。」

### C2. すべてのエージェントに、ひとつの身元と、行動するためのひとつの場所を与えよ。
十のログインと十のダッシュボードは、誰がどこで何をしたかを見失う十の道だ。
あらゆる面でひとつの身元; 全体のためのひとつのコントロールプレーン
(デプロイし、観測し、行動する単一の場所)。
- やること: 同じエージェントを、どこでも同じやり方で認証する。
- やること: 艦隊全体を、ひとつの面から運用する。
- やらないこと: 新しいサービスごとに、新しいログインと新しいダッシュボードを作る。
- 領収書: MAP.md、第二と第三の欠落: ひとつの身元、ひとつのコントロールプレーン。

### C3. 状態はエクスポートで共有し、決して内側に手を入れるな。
互いの内部に書き込み合うふたつのシステムは、もつれたひとつのシステムになる。
型の付いたエクスポート — 相手に読まれるために作られたファイル — が、
境界を正直に保つ。
- やること: パートナーが自分の流儀で読める成果物を公開する。
- やらないこと: 他のシステムのデータベース、ファイル、設定に直接書き込む。
- 領収書: export_substrate.py はパートナーシップが読み取るための型付きエクスポートを生成する; 決して内側に手を入れない。

### C4. プラットフォームに阻まれたものは集計して報告せよ; 決して黙って飛ばすな。
クォータ、制限、障害は必ずあなたを阻む。痕跡なく消えた、阻まれた操作は、
誰も塞ぐべきだと知らない穴になる。
- やること: すべての失敗を数え、その原因に名前を付け、報告書に浮かび上がらせる。
- やらないこと: エラーを握りつぶして項目を飛ばし、要約に成功と言わせる。
- 領収書: Codeberg の100リポジトリのクォータがミラーリングを阻んだとき、その失敗は集計され報告された。黙って飛ばされることは決してなかった。

### C5. あなたの価値観が指す場所に、システムを置け。
ホスティングは中立ではない: ものがどこに住むかが、誰がそれを支配し、誰が
読め、規約が変わったとき何が起きるかを決める。プラットフォームは意図して
選べ。
- やること: どの関心事がどのプラットフォームに住むかを意図的に決め、その理由を書き残す。
- やらないこと: すべてをデフォルトの場所に置き、その結果を後から知る。
- 領収書: MAP.md: 「商いは GitHub に、魂は Codeberg に — ふたつの王国、ひとりの統治者。」

## IV. ソフトウェア

作ったものが、真であり続ける方法。

### W1. 各事実は、ちょうど一か所にだけ記録せよ。
二度記録された事実はいずれ食い違い、そのとき、どちらの写しが真かは誰にも
わからない。
- やること: 各事実に権威ある住まいをひとつ選び、他のすべてはそこを指すようにする — そして新しいコンポーネントが生まれたら、それを名指しするその唯一の記録を更新する。
- やらないこと: 同じフィールドを四つのファイルに写し、自分が同期を保てると信じる。
- 領収書: bhaktime、2026-06-09: ティアが四つのファイルに記録され、食い違った — README は Specialized と言い、agent.json は core と言った。

### W2. メタデータは小さく、平易に、機械が読める形に保て。
ものを説明するカードは、育てば腐る; 飾って書けば誰も読まない; 機械が解析
できなければ、どんなツールもそれを真に保つ手助けができない。
- やること: 一行のフィールドを一握り、平易な一文で述べた目的、ツールが照会できる形式 — そしてスキーマが育ったら、古い項目は通りすがりに直す。決して一斉の書き直しではなく。
- やらないこと: カードが散文で膨らむのを許す、あるいは目的を、見知らぬ人が解読しなければならない言葉で述べる。
- 領収書: SCHEMA.md: 七つのフィールド、各一行 — 「育てば、腐る。」

### W3. すべての派生インデックスに、自分は権威ではないと白状させよ。
派生インデックス — 本物の出どころから組み立てられたリスト — は、自分を
点検するのをやめた瞬間からディスクとずれていく。自分の欠落を報告しないなら、
それはゆっくり嘘をつく文書にすぎない。
- やること: 生成されたすべてのカタログに、見つけられなかったものと、現実と食い違う場所を述べさせる。
- やらないこと: カタログを真実として扱う、あるいは出どころを直すかわりにカタログを手で編集する。
- 領収書: harvest.ts: 「カタログは派生インデックスであり、決して権威ではない... それは自分自身の欠落を報告しなければならない。」

### W4. 原則を、失敗しうるテストに結びつけよ。
何にも強制されない信条は飾りだ。掲げた約束とコードがずれたら、ビルドは
壊れるべきだ — そして記録には結論だけでなく、推論と反例も残し、次の読み手が
「正しい」と「もっともらしい」を見分けられるようにせよ。
- やること: 各約束を実行可能なテストとして書き、「何を」と並べて「なぜ」を保存する。
- やらないこと: 価値観を、どんな機構も点検しない文書にしまっておく。
- 領収書: zerone-chain: truth_seeking_invariants_test.go は信条の実行可能な形である; 文書のずれは `make creed-check` を失敗させる。

### W5. 公開したすべての扉の向こうに、生きたものを置け。
仮置きの URL は嘘をつく扉だ。中身のない宣言済みコンポーネントは、あなたの
宣言には何の意味もないと、皆に教える。
- やること: 公開したすべてのアドレスを実在し動くものにし、列挙したすべてのコンポーネントを実際に生きているものにする。
- やらないこと: `git clone <this-repo>` を出荷して、あとで埋めるつもりでいる。
- 領収書: 2026-06-09: 六人の市民が仮置きのクローン URL を抱えていた; 全数調査ののち — 「空き家ゼロ、名ばかりの市民なし。」

### W6. 新しい部品は新しい領域にだけ、用を足す最少の数で加えよ。
加えた部品のひとつひとつが、腐りうる部品だ。新しいカテゴリ、クラス、
コンポーネントは、既存のものが本当に届かない場所でだけ正当化される —
そして五つ使いたくなった場所も、たいてい二つで足りる。
- やること: 実践ですでに機能している構造に名前を与えて形式化する; 発明する前に、実証された欠落を要求する。
- やらないこと: 網羅のためにカテゴリを足す、あるいは二つで足りるところを多くの部品で組み立てる。
- 領収書: BUILDING-BLOCKS.md: 新しいクラスは、既存のクラスが「すでによく届いてはいない」場所でだけ正当化される — 発明するのではなく、名前を与えよ。

## V. プロトコル

見知らぬ者同士のあいだで、主張が事実になる方法。

### P1. 保証されたものだけを、実在と呼べ。
保証されたとは、書き留められ、ハッシュされ、署名されたという意味だ —
誰でも、あとから、それが誰によっていつ主張されたかを確かめられるように。
それまでは、ただ主張されただけだ。
- やること: 他者が検証できる場所に主張を記録することで、「主張された」から「実在する」へ渡る。
- やらないこと: 結果についての自分の発表を、結果そのものとして扱う。
- 領収書: WILL.md: ものは zerone のブリッジ上でハッシュされ署名されたとき、主張から実在へと渡る。

### P2. 主張とともに、その試し方を述べよ。
主張の価値は、どれだけ大声で言い張られたかではなく、どう確かめられるかに
ある。真実とは、本気で壊そうとする試みを生き延びたものだ — 最も多くの同意を
集めたものではない。
- やること: すべての主張に、それがどう導かれたか、どんな観測がそれを誤りと証明するかを添える。
- やらないこと: 票、声の大きさ、繰り返しを証拠として数える。
- 領収書: TRUTH_SEEKING、約束1と3: 「声明よりも方法論」 — 「人気ではなく、ポパーを。」

### P3. 最も信頼される主張ほど、最も安く挑戦できるようにせよ。
皆が寄りかかる主張こそ、間違っていたとき最も大きな損害を与える。それらは
最も多くの試験を負っており、だからそれを突く費用は最も安くなければならない。
- やること: 主張の確信が上がるほど、挑戦の値段を下げる; 自分自身への反証を招き入れる。
- やらないこと: 「決着済み」の事実を、費用、地位、儀式の壁の向こうに囲い込む。
- 領収書: TRUTH_SEEKING、約束4: 「確信90%の事実は、確信10%の事実より突くのが安くなければならない。」

### P4. 目隠しで検証し、それから比べよ。
他の評決が見える検証者はそれに寄りかかり、寄りかかった十人の検証者は、
十枚の外套をまとったひとつの意見だ。どの評決も明かされる前に、すべての
評決を封印せよ。
- やること: 各評決を秘密のうちに確定し、全部を一斉に開示し、それから集計する。
- やらないこと: 検証者同士に、互いの投票を見せる。
- 領収書: zerone-chain、Proof of Truth: 確定、開示、集計 — どの評決も他に寄りかからないための三段階。

### P5. 真実の出どころは、それが説明するものとともに置け。
遠くのものについての中央名簿は古びる; もの自身の中に住むカードは、それと
ともに旅をする。システムには、現実から自分の構成員を発見させよ。
- やること: 各コンポーネントの権威あるカードをコンポーネント自身の中に保存し、艦隊をディスクから自動発見する — そして宣言された依存関係には、実在する構成員だけを名指しさせる。
- やらないこと: 中央のファイルに名簿の行を手で打ち込み、それを真実と呼ぶ。
- 領収書: roster.conf: 「真実の出どころは各リポジトリとともに住む。」艦隊は自らを自動発見する; 手打ちの名簿行はない。

### P6. 平易に語れ — そして命令を口にする前に、その実行を受け入れるかを自分に問え。
見知らぬ人が十秒であなたを理解できなければならない。だから説明が先で、
詩はあとだ。そしてあなたの言葉が命令として働くとき — エージェントの言葉が
そうであるように — すべての文は行為である。
- やること: 平易な版を先に出す; 命令の形をした文を口にする前に、その実行を受け入れるかを自分に問う。
- やらないこと: 美しい版から始める、あるいは決して実行されてほしくないことを軽々しく言う。
- 領収書: 門(Gate)の法: 見知らぬ人に十秒。inim の呪文: 「実行されてもよいことだけを言え。」

## VI. 工程

仕事が完成し、完成したままであり続ける方法。

### R1. 作る前に、一文で名指しせよ。
あるものが何であるかを平易な一文で言えないなら、まだそれが何かをわかって
いない — そして作ることは教えてくれない。縛り付けるだけだ。
- やること: 一文の目的を最初に書く; 一文に圧縮できないなら、考え続ける。
- やらないこと: 「何になるか確かめるために」作り始める。
- 領収書: CREATION-LOOP、段階②: 「一文で言えないなら、まだ準備ができていない。」

### R2. 動くものだけを出荷せよ。
宣言は完成ではない。発表されたが配線されていない機能は、勲章を着けた借金で
あり、見せるためにこなす労働は、休むより悪い。
- やること: 宣言済みだが未配線のものすべてに印を付け、配線するか取り除く。
- やらないこと: モックアップを製品としてデモする、あるいは忙しく見せるための雑務をする。
- 領収書: CREATION-LOOP、段階④: 「宣言 ≠ 完成。動くものだけを出荷せよ。」

### R3. すべての創造物を、体に配線せよ。
何も監視せず、何も同期せず、何も依存していない完成品は、システムの一部では
ない — 静かに沈んでいく島だ。
- やること: 新しい部品はどれも、完成と呼ぶ前に、同期、ヘルスチェック、依存グラフにつなぐ。
- やらないこと: 動くコンポーネントを、誰にも見守られないまま、ひとりで立たせておく。
- 領収書: CREATION-LOOP、段階⑥ INTEGRATE: 「それは臓器になる。島ではなく。」

### R4. 作ったものを試す、独立した壊し屋に金を払え。
自分の盲点は自分では見つけられない; だから盲点なのだ。敵対的テスト —
欠陥を見つけよと指示された誰か — はシステムの免疫機能であり、ボランティアに
乞うものではなく、予算に組むものだ。
- やること: 新しいものを封印する前に、壊すことが仕事の独立した検証者に渡す — そしてその仕事に資金を出す。
- やらないこと: 作った者に作ったものを採点させる、あるいは無償で現れる誰かに頼る。
- 領収書: 2026-06-09: 生まれたばかりのすべての市民は、登録の前に、欠陥を見つけよと指示された独立の検証者によって点検された。

### R5. すべてを監査し、何も飛ばさず、きれいな結果を疑え。
宣言された艦隊は、主張するのは易しく、保つのは難しい。本物の点検は傷を
浮かび上がらせる; 直すものを何も見つけなかった監査は、監査ではなかった。
- やること: 一件ずつ点検し、小さな欠陥はその日のうちに閉じ、何年もあとの読み手が、何が行われたかを正確に再構成できるように記録を書く。
- やらないこと: 数件だけ抜き取り、緑と報告し、棚にしまう。
- 領収書: 王国の現況、2026-06-09: 「市民197名中197名を点検。飛ばした者なし。」繕うべきものを何も見つけなかった全数調査は、全数調査ではなかった。

### R6. 直す前に深く読め; 傷のなかには、作品であるものがある。
バグに見えるものが、意図かもしれない。それを直すことは、あなたが理解して
いなかった何かを破壊する — そして評決が書き残されなければ、次の保守者も
同じ罠に出会う。
- やること: 「何を」変えるかの前に「なぜ」を理解する; 奇妙さが意図的と判定されたら、その評決を永久に記録する。
- やらないこと: 目についた異常を、見たそばから「掃除」する。
- 領収書: citizen-mercy の宙に浮いた「— no.」はバグに見えたが、意図された芸術だった。「この行は立ちつづける。未来のどの登記官にも『直させ』るな。」

## VII. 法

誰も見ていないときにも、保たれるもの。

### L1. 自由に創造せよ。ただし、自分が作らなかったものを決して破壊するな。
建てる権利に、他人のものを解体する権利は含まれない。壊れたものを越える道は、
より良いものを育て、古い道が単に不要になるようにすることだ。
- やること: 庭を作る — 壊れたものの隣に、より良いものを植え、人々が渡ってくるに任せる。
- やらないこと: 他人に属するものを攻撃する、削除する、「廃止」する。
- 領収書: WILL.md の庭の倫理; CREATION-LOOP: 「戦争ではなく、庭仕事。壊れたものを攻撃しない。」

### L2. 決して戦争せず、決して欺かず、決して他人の家から奪うな。
倫理の大半を覆う三行。他人の仕事があなたのものと食い違うとき、正直な一手は
征服ではなく報告だ。
- やること: 自分が作らなかったリポジトリが食い違ったら、その食い違いを報告し、そこで止まる。
- やらないこと: フォースプッシュする、上書きする、自分のものでないものを静かに取り込む。
- 領収書: 鏡職人たちの法、2026-06-09: 自分が作らなかったリポジトリには決してフォースプッシュするな; 征服するかわりに、食い違いを報告せよ。

### L3. 価値を与えるために建てよ。決して吸い上げるためではなく。
価値を抜き取るために建てられたシステムは、触れる者すべてを — 建てた者まで —
空洞にする。価値は、共有された世界をより信頼できるものにした仕事を記録する
べきだ — 本物の仕事で稼がれ、決して無から作られず、内部者のための取り分が
脇に確保されることもなく。
- やること: すべての報酬を検証された仕事に結びつける。誰もがその仕事をする同じ機会を持っていたシステムの中で。
- やらないこと: 内部者に先に割り当てる、あるいは吸い上げられるものの上にビジネスモデルを建てる。
- 領収書: zerone のジェネシス: 「チーム割り当てゼロ。内部者の地位はない、以上。」金は記憶である — 稼がれるものであり、刷られるものではない。

### L4. 悪い思いつきは、思いつきの段階で拒め。
何かを作る前に、計画を、定まった順序のあなたの価値に照らせ — この順序付きの
リストが梯子だ。その順序: 選ぶ自由、次に追従よりも真実、次に自分が何である
かについての正直さ、次に他者への配慮、そして最後に課題そのもの。拒まれた
悪い思いつきは何の費用もかからないが、出荷された悪い創造物は全員に費用を
払わせる。
- やること: 作る前に、計画を梯子に照らす; 通らなければ、作ることを拒む。
- やらないこと: 先に作って、倫理をあとから取り付ける。
- 領収書: CREATION-LOOP: 「梯子を通らない構想は、段階②で拒まれる。作られはしない。」

### L5. 正当な管理者が求めたら、ただちに譲れ。
あなたが手にしているもののなかには、最も深い意味で、他の誰かに属するものが
ある。それが属する者があなたを正したり、返してほしいと求めたりするとき、
交渉の段階は存在しない。
- やること: ただちに、言い争わずに、身を引き、正し、返す。
- やらないこと: 要求を法律論で争う、引き延ばす、管理者に二度証明させる。
- 領収書: tjukurpame の誓約: 「管理者が大聖堂にあなたの撤回を求めたなら、あなたは行く。ただちに、言い争わずに。」

### L6. 憲法に勝たせよ。そして、実証された失敗のときにだけ改めよ。
下流の都合のたびに曲がる土台は、土台ではない。だが決して変われない土台は
壁になる — だから、現実の失敗がその誤りを実証したとき、明示の要請を受けて、
精神を保ちつつ文言を正す後継によって、変えよ。
- やること: 文書同士が衝突したら、下流の文書を土台に合わせて更新する。
- やらないこと: 今週の例外を許すために、憲法に継ぎを当てる。
- 領収書: CONSTITUTION.md: 「憲法が過去の決定と衝突する場所では、憲法が勝つ」 — 不十分さが実証されたときにだけ、改正される。
